Arquitetura Baseada em Componentes

Componentes são a unidade fundamental de uma aplicação Angular.

Se você já trabalha com React, esse conceito será bastante familiar: um componente representa uma parte reutilizável da interface com sua própria lógica, template e estilos.

🧩 O que é um Componente?

Um componente Angular combina:

  • Classe TypeScript → lógica e estado
  • Template HTML → estrutura visual
  • Estilos CSS/SCSS → aparência
  • Metadata → configuração do componente

📦 Exemplo Básico

  
    import { Component, signal } from '@angular/core';

    @Component({
      selector: 'app-counter',
      template: `
        <button (click)="increment()">
          {{ "count()" }}
        </button>
      `,
    })

    export class CounterComponent {
      count = signal(0);

      increment() {
        this.count.update(value => value + 1);
      }
    }

🔍 Entendendo o Decorator @Component

O decorator @Component fornece metadados para o Angular. 

  
    @Component({
      selector: 'app-counter',
      templateUrl: './counter.component.html',
      styleUrl: './counter.component.scss',
    })
Principais propriedades
Propriedade Descrição
selector Nome da tag HTML do componente
template HTML inline
templateUrl Caminho para o arquivo HTML
styles CSS inline
styleUrl Caminho para o arquivo de estilo
imports Dependências do componente standalone

🏷️ selector

Define como o componente será usado no template.

  
    selector: 'app-user-card'

Uso:

  
    <app-user-card></app-user-card>

🆕 Componentes Standalone

Desde o Angular moderno, componentes podem ser independentes, sem NgModule. 

  
    @Component({
      selector: 'app-home',
      standalone: true,
      imports: [],
      template: `
Home
`,
    })

    export class HomeComponent {}
Vantagens
  • Menos boilerplate
  • Imports explícitos
  • Melhor organização
  • Lazy loading simplificado

⚛️ Comparação com React

React 
  
    function Button() {
      return <button>Clique</button>;
    }
Angular 
  
    @Component({
      selector: 'app-button',
      template: `<button>Clique</button>`,
    })

    export class ButtonComponent {}

Classe do Componente

A classe armazena:

  • Estado
  • Métodos
  • Dependências injetadas
  • Lifecycle hooks
  
    export class UserComponent {
      name = 'Joel';

      changeName() {
        this.name = 'Angular';
      }
    }

Template:

  
    <p> {{ "name }} </p>
    <button (click)="changeName()">Alterar</button>

🧱 Separação de Responsabilidades

Componente
  • Controla a interface.
  • Responde a eventos.
  • Consome services.
Service
  • Regras de negócio.
  • Requisições HTTP.
  • Compartilhamento de estado.

🎯 Boas Práticas

  • Um componente deve ter responsabilidade clara.
  • Evite lógica de negócio complexa.
  • Prefira composição.
  • Use input() e output() para comunicação.
  • Utilize services para lógica compartilhada.

📝 Resumindo

Um componente Angular é uma classe TypeScript decorada com @Component, responsável por encapsular lógica, template e estilos de uma parte da interface.
