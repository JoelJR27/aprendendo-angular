O que é Angular e sua filosofia

Se você vem do React, a primeira mudança importante é entender que o Angular não é apenas uma biblioteca para construir interfaces. Ele é um framework completo, com decisões arquiteturais já definidas.

📌 Definição

Angular é um framework frontend mantido pela Google para criação de aplicações web escaláveis, utilizando TypeScript como linguagem principal.

Ele já oferece, de forma nativa:

  • Sistema de componentes
  • Roteamento
  • Injeção de dependência
  • Consumo de APIs HTTP
  • Formulários
  • Testes
  • Gerenciamento reativo com Signals

🧠 Filosofia do Angular

O Angular segue uma filosofia opinionada.

Isso significa que o framework já define:

  • Como organizar pastas
  • Como criar componentes
  • Como compartilhar lógica
  • Como consumir APIs
  • Como navegar entre páginas
React vs Angular

No React, você escolhe as ferramentas:

  • React Router
  • Axios
  • Zustand / Redux
  • React Hook Form

No Angular, tudo isso já vem integrado ao framework.

🏗️ Analogia Prática

React

É como receber peças soltas e montar sua própria casa.

Angular

É como receber uma casa com fundação, encanamento e elétrica já prontos.

Conceitos Centrais do Angular

  1. Componentes

    -

    Cada tela ou parte da interface é um componente.

  2. Templates

    -

    O HTML possui sintaxe própria do Angular.

  3. Services

    -

    Classes responsáveis por lógica de negócio e acesso a dados.

  4. Dependency Injection

    -

    O Angular cria e injeta dependências automaticamente.

  5. Signals

    -

    Estado reativo moderno do Angular.

  6. RxJS

    -

    Sistema poderoso de programação reativa.

  7. Router

    -

    Sistema de rotas nativo.

📂 Exemplo Mental de Estrutura

  • Component → Interface
  • Service → Regras de negócio
  • HttpClient → API
  • Signal → Estado local
  • Router → Navegação

🔥 Exemplo Comparativo

React 
  
      function Counter() {
        const [count, setCount] = useState(0)

        return (
          <button onClick={() => setCount(count + 1)}>
            {count}
          </button>
        )
      }
Angular 
    
  @Component({
    selector: 'app-counter',
    template: `
      <button (click)="increment()">
        {{ "count()" }}
      </button>
    `
  })

  export class CounterComponent {
    count = signal(0)

    increment() {
      this.count.update(value => value + 1)
    }
  }

Mudança de Mentalidade Mais Importante

No React, é comum concentrar muita lógica dentro do componente.

No Angular, o componente deve ser enxuto e delegar responsabilidades para services.

Regra prática
  • Component = apresentação e interação
  • Service = lógica e dados

Por que Empresas Escolhem Angular?

  • Padronização
  • Escalabilidade
  • Forte tipagem
  • Excelente suporte para aplicações corporativas
  • Arquitetura consistente entre equipes

O Que Você Deve Absorver Neste Primeiro Conceito

Ao estudar Angular, internalize que:

  1. Angular é um framework completo.
  2. O framework define convenções.
  3. TypeScript é obrigatório.
  4. Dependency Injection é central.
  5. Services são fundamentais.
  6. Components devem ser simples.
  7. Signals são o equivalente moderno ao useState.

📝Resumindo em Uma Frase

Angular é um framework opinionado que oferece uma arquitetura completa para construção de aplicações escaláveis, utilizando TypeScript, componentes, services e dependency injection.
