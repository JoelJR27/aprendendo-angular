Templates HTML e Data Binding

Se componentes são o cérebro da aplicação, os templates são a camada visual.

No Angular, o template é um arquivo HTML com extensões especiais que permitem:

  • Exibir dados da classe.
  • Escutar eventos.
  • Atualizar propriedades.
  • Renderizar conteúdo condicionalmente.
  • Iterar listas.

Se você vem do React, pense no template Angular como uma alternativa ao JSX.

🧠 O que é Data Binding?

Data binding é a conexão entre:

  • A classe TypeScript do componente.
  • O template HTML.
   (TypeScript) <----> Template (HTML)

📌 Os 4 Tipos de Binding

  1. Interpolation
  2. Property Binding
  3. Event Binding
  4. Two-Way Binding

1. Interpolation { { } }

Exibe valores da classe no HTML.

Componente

  
    export class UserComponent {
      name = 'Joel';
    }

Template

  
    <h1>Olá, { { 'name' } }!</h1>
Resultado 
  
    <h1>Olá, Joel!</h1>
Exemplo:
Olá, {{ name }}!

2. Property Binding [property]

Define propriedades do elemento HTML.

Componente

  
    export class ImageComponent {
      imageUrl = 'path/to/image.jpg';
    }

Template

  
    <img [src]="imageUrl" />

3. Event Binding (event)

Escuta eventos do DOM.

Componente

  
    export class CounterComponent {
      count = 0;

      increment() {
        this.count++;
      }
    }

Template

  
    <button (click)="increment()">Incrementar</button>
    <p>Contagem: { { count } }</p>
Exemplo:

Contagem: {{ count }}

4. Two-Way Binding [(ngModel)]

Sincroniza input e propriedade.

Componente

  
    export class UserComponent {
      name = '';
    }

Importante

Para usar [(ngModel)], o módulo precisa importar FormsModule. 

  
    import { FormsModule } from '@angular/forms';

  @NgModule({
    imports: [FormsModule]
  })

  export class AppModule { }

⚛️ Comparação com React

React

  
    <input value={name} onChange={(e) => setName(e.target.value)} />

Angular

  
    <input [(ngModel)]="name" />
